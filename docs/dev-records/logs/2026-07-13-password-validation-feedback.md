# 开发记录：密码校验中文提示

## 完成内容

用户管理页面的新增用户和重置密码入口现在会实时显示中文密码校验状态。输入不符合规则时，会明确指出长度不足、缺少字母或缺少数字；满足要求后显示“密码要求已满足”。提交时仍会执行统一校验，避免无效请求发送到后端。

## 实际改动文件

* `b_k3ewYvsOEc1/src/lib/userValidation.js`：新增密码实时状态函数，按后端规则判断有效性并返回具体中文提示。
* `b_k3ewYvsOEc1/src/components/UserManagement.vue`：新增用户和重置密码表单接入实时状态，增加错误/通过样式及无障碍提示属性。
* `tests/user-create-validation.test.ts`：覆盖空密码、长度不足、首尾空格、缺少字母、缺少数字和有效密码。
* `docs/dev-records/plans/2026-07-13-password-validation-feedback.md`：记录本次范围和验证方案。

## 关键行为决策

* 前端和后端保持同一规则：去除首尾空格后的长度至少为 8，并同时包含字母和数字。
* 页面提示只解释现有安全规则，不降低或扩展密码策略。
* 创建用户和重置密码共用同一校验函数，避免两个入口提示不一致。
* 保留后端校验作为最终安全边界，前端校验只负责即时反馈和减少无效请求。

## 已知风险

* 后端错误文案仍是英文，但正常页面操作会先显示中文前端提示。
* 浏览器自动填充密码后，提示状态取决于浏览器是否触发输入更新；用户手动输入和粘贴均正常响应。

## 后续排查入口

如果密码提示与接口结果不一致，优先检查：

1. `userPasswordValidationState` 是否仍与后端 `validatePasswordStrength` 保持一致。
2. 新增用户和重置密码输入框是否都引用同一个状态函数。
3. 浏览器 Network 中 `/api/users` 或 `/api/users/:id/password` 的响应正文。
4. 线上是否已部署包含本次前端资源的版本。
