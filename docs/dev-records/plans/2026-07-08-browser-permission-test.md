# 开发计划：browser-permission-test

## 背景

需要使用浏览器实际验证账号权限模块在页面上的表现，确认不同权限账号登录后模块入口展示、无模块空状态、admin 管理入口等行为是否符合预期。

## 本次目标

使用 Chrome 对线上或本地页面进行人工路径验证，重点检查模块入口显示和账号权限相关页面行为。

## 本次不做

* 不修改业务代码。
* 不修改数据库结构。
* 不调整登录逻辑。
* 不部署新版本。
* 不创建或删除生产用户，除非已有页面验证必须且用户明确允许。

## 涉及范围

* 前端页面：登录、顶部模块导航、用户管理/角色管理入口。
* 后端接口：登录接口、当前用户信息接口、权限相关数据接口。
* 测试文件：本次优先做浏览器手动验证，不新增自动化测试。

## 开发步骤

1. 打开线上页面。
2. 检查当前登录状态和可见模块入口。
3. 使用可用账号登录验证 admin modules 和管理入口。
4. 检查用户管理/角色管理页面是否只展示业务模块。
5. 如能切换或创建测试用户，再验证不同模块账号的页面入口。
6. 将测试结果写入开发记录。

## 验证方式

* 使用 Chrome 打开线上地址。
* 通过页面登录和导航进行手动验证。
* 记录截图/观察结果。
* 如遇接口或账号限制，记录无法验证的部分和原因。
