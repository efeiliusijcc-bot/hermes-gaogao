# 开发计划：live-module-usage-test

## 背景

需要验证线上环境中除 QA 问答外的业务模块是否真的可用，尤其是 AI 智能体深度编报是否可以创建编报任务，而不仅仅是页面可进入。

## 本次目标

在 Chrome 线上页面中验证非 QA 模块可用性，重点实际创建一条编报任务，并记录任务创建结果。

## 本次不做

* 不测试 QA 问答。
* 不修改业务代码。
* 不修改数据库结构。
* 不调整权限配置。
* 不删除线上业务数据。

## 涉及范围

* 前端页面：线上 `https://hermes-gaogao.vercel.app/`
* 业务模块：AI 智能体深度编报、每日动态感知、拟稿助手
* 测试记录：`docs/dev-records/logs/2026-07-09-live-module-usage-test.md`

## 开发步骤

1. 使用 Chrome 打开或认领线上 Hermes 页面。
2. 确认当前登录账号和可见业务模块。
3. 切换到 AI 智能体深度编报，填写测试标题和背景。
4. 触发编报规划生成，并在规划确认后提交正式编报任务。
5. 记录任务是否创建成功、页面是否进入生成或详情状态。
6. 只做 Daily Awareness 和 Draft Assistant 的页面级可用性复核，不触发额外生成，除非用户后续明确要求。

## 验证方式

* Chrome 手动/自动化页面验证。
* 检查编报页面是否出现新任务、生成状态或任务详情。
* 不运行 QA 问答相关测试。
