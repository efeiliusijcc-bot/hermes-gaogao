# 开发计划：non-admin-login-live-test

## 背景

需要使用非 admin 账号登录线上系统，验证 operator/viewer 等不同角色在真实页面中的模块可见性和使用限制。

## 本次目标

尽量使用现有测试账号登录线上页面，分别验证操作员和观察员账号的模块入口、编报创建权限、用户管理入口和受限行为。

## 本次不做

* 不擅自重置现有线上用户密码。
* 不擅自创建临时测试用户，除非用户明确授权。
* 不删除线上数据。
* 不修改权限配置、数据库结构或业务代码。
* 不测试 QA 问答真实发送。

## 涉及范围

* 线上页面：`https://hermes-gaogao.vercel.app/`
* 用户角色：operator、viewer
* 项目记录：`docs/dev-records/logs/2026-07-09-non-admin-login-live-test.md`

## 开发步骤

1. 检查项目中是否记录了可用测试账号密码。
2. 若找到现成测试账号，使用 Chrome 登录线上页面。
3. 验证 operator 是否可进入业务模块并创建编报规划/任务。
4. 验证 viewer 是否不能创建编报、不能进入用户管理。
5. 记录每个账号的测试结果。
6. 若没有可用密码，记录阻塞原因，并等待用户提供密码或授权创建临时账号。

## 验证方式

* Chrome 线上登录验证。
* 页面入口检查。
* 编报创建权限检查。
* 用户管理入口检查。
