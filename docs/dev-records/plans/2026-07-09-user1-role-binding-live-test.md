# 开发计划：user1-role-binding-live-test

## 背景

需要使用线上账号 `user1` 验证“用户绑定不同角色后，页面模块权限和写入权限是否随角色变化”。用户已提供 `user1` 密码，并允许通过绑定不同角色测试效果。

## 本次目标

使用 admin 管理端临时调整 `user1` 角色，再用 `user1` 登录验证：

* 观察员角色的模块可见性和写入限制。
* 操作员角色的模块可见性和写入能力。
* 必要时验证管理员角色入口，但避免长期保留 admin 权限。
* 测试完成后尽量恢复 `user1` 原始角色配置。

## 本次不做

* 不测试 QA 真实发送。
* 不删除线上业务数据。
* 不重置 `user1` 密码。
* 不修改后端代码、数据库结构或权限映射规则。
* 不长期保留测试期间临时授予的高权限。

## 涉及范围

* 线上页面：`https://hermes-gaogao.vercel.app/`
* 测试账号：`user1`
* 管理账号：当前 Chrome 中的 Administrator 会话
* 页面：用户管理、AI 智能体深度编报、每日动态感知、拟稿助手
* 记录：`docs/dev-records/logs/2026-07-09-user1-role-binding-live-test.md`

## 开发步骤

1. 在 admin 会话中记录 `user1` 当前角色和模块。
2. 将 `user1` 设置为观察员，登录验证模块入口和写入按钮状态。
3. 将 `user1` 设置为操作员，登录验证编报创建和写入能力。
4. 如有必要，将 `user1` 临时设置为管理员，验证用户管理入口和管理权限显示。
5. 测试完成后恢复 `user1` 原始角色。
6. 将实际结果写入开发记录。

## 验证方式

* Chrome 页面操作验证。
* 登录 `user1 / 123456`。
* 检查模块入口是否按角色变化。
* 检查 viewer 是否不能创建编报。
* 检查 operator 是否可以创建编报任务。
* 检查用户管理入口是否只对具备管理权限的角色显示。
