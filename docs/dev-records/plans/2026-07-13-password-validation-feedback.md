# 开发计划：密码校验中文提示

## 背景

创建用户和重置密码已经有基础密码规则，但用户输入不符合要求时，页面只显示固定规则或依赖浏览器原生提示，不能明确指出当前缺少什么。需要把密码要求改成实时、具体、可理解的中文反馈。

## 本次目标

* 密码输入时实时显示当前校验状态。
* 分别提示长度不足、缺少字母、缺少数字。
* 不符合要求时使用红色错误状态，符合要求时显示通过状态。
* 提交创建用户或重置密码时继续执行同一套校验，并在页面错误区域显示中文原因。

## 本次不做

* 不修改后端密码强度规则。
* 不修改用户创建和重置密码接口格式。
* 不修改登录流程、RBAC、数据库和其他业务模块。
* 不增加密码复杂度等级或特殊字符要求。

## 涉及范围

* 前端页面：`b_k3ewYvsOEc1/src/components/UserManagement.vue`
* 校验工具：`b_k3ewYvsOEc1/src/lib/userValidation.js`
* 测试文件：`tests/user-create-validation.test.ts`
* 开发记录：`docs/dev-records/logs/2026-07-13-password-validation-feedback.md`

## 开发步骤

1. 先扩展测试，定义长度不足、缺少字母、缺少数字和通过四种状态，并运行确认测试失败。
2. 在校验工具中新增可复用的密码状态函数，保持现有提交校验函数兼容。
3. 在新增用户和重置密码表单中接入实时状态，设置错误样式和 `aria` 属性。
4. 补充提示区域样式，确保普通提示、错误和通过状态清楚但不过度抢眼。
5. 运行专项测试和前端构建，并进行页面检查。

## 验证方式

* `npx tsx tests/user-create-validation.test.ts`
* `npx pnpm@9.15.9 --dir b_k3ewYvsOEc1 build`
* 页面手动验证：管理员进入“用户管理”，在新增用户密码框和重置密码框分别输入 `123456`、`abcdefgh`、`12345678`、`user1234`。
* 预期：前三种分别提示具体缺项；`user1234` 显示密码要求已满足；无效密码不能提交请求。
