# 开发计划：user-create-400

## 背景

线上用户管理页面调用 `POST /api/users` 返回 400，导致管理员无法创建用户。需要确认是前端 payload、后端字段校验、角色绑定、用户名重复还是数据库约束错误。

## 本次目标

定位线上 400 的明确错误原因，修复用户创建主流程，并确保错误信息能在前端清楚展示。

## 本次不做

* 不重构登录和 JWT 主流程。
* 不修改 RBAC 表结构，除非确认现有结构缺失。
* 不修改报告、QA、Draft Assistant、Daily Awareness。
* 不清理工作区其他未提交改动。

## 涉及范围

* `b_k3ewYvsOEc1/src/components/UserManagement.vue`
* `b_k3ewYvsOEc1/src/lib/api.js`
* `server/users.controller.ts`
* `server/users.service.ts`
* 用户/角色相关测试
* 云节点 `hermes-api` 日志和线上接口响应

## 开发步骤

1. 检查前端创建用户表单和请求 payload。
2. 检查后端 create 校验、角色解析和数据库写入逻辑。
3. 查看线上 API 日志及 400 响应 body。
4. 用管理员 token 复现一次不产生脏数据的校验请求。
5. 先补失败测试，再做最小修复。
6. 运行后端/前端构建和权限回归测试。
7. 写入开发记录，并说明部署要求。

* 线上接口验证：合法用户创建成功，字段缺失和重复用户名返回明确错误。
