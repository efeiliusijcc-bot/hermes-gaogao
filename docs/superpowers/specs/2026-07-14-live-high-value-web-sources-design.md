# 编报过程高价值联网信源展示设计

## 目标

在不改变现有编报流程、接口入口和页面布局的前提下，让信源面板在编报运行过程中同时展示：

- 数据库召回信源；
- 联网搜索采集的高价值信源。

每条信源必须明确标注来源分类，用户可以在“全部”视图中混合查看，也可以继续使用现有的“数据库检索工具”和“互联网搜索工具”分类查看。

## 范围

本次仅修改 Hermes 后端信源聚合和现有前端信源面板的刷新行为。

不修改：

- 深度编报任务执行顺序；
- Hermes 调研子任务及 Tavily 搜索逻辑；
- 数据库召回、混合检索和 RRF 逻辑；
- 报告生成、质量检查及引用编号逻辑；
- `gaogao-api` 容器；
- 现有页面布局和视觉主题。

## 当前问题

`ReportsService.toolSearchSources()` 当前从 API 本地 artifact 根目录读取 `research` 产物。运行中的调研子任务则把 `research_*.json` 写入 Hermes 共享目录，即 `hermes-inbox/<jobId>/research`。两处目录尚未同步时，`/api/report-jobs/:jobId/sources` 返回的 `toolSearchCount` 为 0，即使调研子任务已经产生 Tavily 信源。

前端已经具备“全部”“数据库检索工具”“互联网搜索工具”等分类和对应卡片，不需要新增页面结构，但信源页签打开后不会自动刷新。

## 后端设计

### 双目录读取

`toolSearchSources(job)` 同时读取：

1. API 本地 artifact 目录：`<artifactRoot>/<jobId>/research`；
2. `resolveHermesJobDir(job)` 对应的共享任务目录：`<hermesJobDir>/research`。

每个目录读取：

- `consolidated.json`；
- 符合 `research_[a-z0-9_-]+.json` 的文件。

目录不存在、文件尚未生成、文件被并发写入或 JSON 暂时无法解析时返回当前可用结果，不影响任务主流程。

### 高价值筛选

联网信源满足以下任一条件时展示：

1. URL 被 `evidence_cards` 引用；
2. `credibility_score >= 0.8`；
3. `credibility_tier` 为 `high` 或 `medium-high`。

没有 URL 的记录不进入信源列表。普通搜索候选、Wikipedia、YouTube、搜索结果页等未达到上述条件的来源不展示。

### 去重与分类

联网信源按规范化 URL 去重：

- 移除 URL fragment；
- 移除常见跟踪参数；
- 域名转小写；
- 保留影响正文定位的必要路径和业务参数。

同一规范化 URL 出现多次时，优先保留：

1. 被证据卡片引用的记录；
2. 可信度更高的记录；
3. 内容字段更完整的记录。

数据库和联网信源在各自渠道内去重。同一 URL 若同时来自数据库和联网搜索，可以各保留一条，以免丢失来源渠道；在“全部”视图中分别标注：

- `sourceGroup=database_recall`，显示“数据库召回”；
- `sourceGroup=tool_search`，显示“联网搜索采集”。

联网搜索最多返回 50 条高价值信源，沿用现有分页接口和响应结构。

## 前端设计

复用现有信源面板、分类卡片、筛选器和列表，不新增页面区域。

### 展示

- “全部”视图同时展示数据库召回和联网搜索采集信源；
- “数据库检索工具”只展示数据库召回；
- “互联网搜索工具”只展示高价值联网信源；
- 每条卡片使用现有 `sourceGroup/sourceOrigin` 显示清晰分类文字；
- 联网信源保留标题、网站、发布时间、摘要、相关性和 URL。

### 自动刷新

当以下条件同时满足时，每 5 秒刷新当前信源列表第一页：

- 当前页签为“信源”；
- 当前任务状态为 `queued` 或 `running`；
- 当前任务存在 `jobId`。

切换页签、任务完成/失败、切换任务或组件卸载时停止定时器。自动刷新不覆盖用户当前筛选条件；请求重叠时沿用现有 request id 机制丢弃旧响应。

手动“刷新”按钮继续保留。

## 数据流

```text
Hermes research_*.json / consolidated.json
                    +
API artifact research 目录
                    |
                    v
提取来源 -> 高价值筛选 -> 规范化 URL 去重 -> tool_search
                    |
数据库召回 --------+--------> /report-jobs/:jobId/sources
                                      |
                                      v
                           现有信源面板每 5 秒刷新
```

## 错误处理

- 任一目录不可读时继续读取另一目录；
- 单个 JSON 文件非法时跳过该文件；
- 联网信源读取失败时仍返回数据库信源；
- 前端单次刷新失败时保留上一轮数据，不清空列表；
- 不向页面显示文件路径、解析异常或敏感原文。

## 测试清单

### 后端

1. artifact 目录中的高可信来源可以返回；
2. Hermes 共享目录中的 `research_A.json` 可以在任务运行中返回；
3. 被 evidence card 引用但没有可信度标签的来源可以返回；
4. 低可信普通候选不会返回；
5. 同一联网 URL 的跟踪参数变体只返回一条；
6. 数据库和联网来源同时出现在 `type=all`，且分类正确；
7. 缺失目录和非法 JSON 不影响数据库来源响应；
8. 权限与 owner 隔离保持不变。

### 前端

1. “全部”视图同时显示两类来源；
2. 卡片分别标注“数据库召回”和“联网搜索采集”；
3. 运行中且信源页签可见时每 5 秒刷新；
4. 任务结束、离开信源页签和组件卸载后停止刷新；
5. 刷新失败保留已有数据；
6. 现有筛选、分页、展开、复制和导入操作不回归。
