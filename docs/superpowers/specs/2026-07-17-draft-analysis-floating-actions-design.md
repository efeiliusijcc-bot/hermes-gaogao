# 拟稿助手事件分析页悬浮操作栏设计

## 目标

将事件分析页底部的“返回重新输入”和“生成提纲”改为统一悬浮操作栏，让用户查看长分析内容时随时执行下一步操作。

本次仅调整前端布局与响应式样式，不修改分析、重试、生成提纲流程或后端接口。

## 布局

- 操作栏固定在视口底部，最大宽度与事件分析内容区一致，为 `840px`。
- “返回重新输入”位于左侧，“生成提纲”位于右侧。
- 桌面端沿用白色半透明背景、边框、阴影、圆角和模糊效果。
- 移动端两个按钮保持双列同行，文字允许换行但不得溢出。
- 内容区增加底部留白，最后一段分析可以滚动到悬浮栏上方。

## 状态与交互

- “返回重新输入”继续触发 `back` 事件。
- “生成提纲”继续触发 `generate` 事件。
- `loading`、`generating` 和 `error` 对按钮的禁用规则保持不变。
- “正在生成”文案与加载图标行为保持不变。

## 组件范围

- 修改 `DraftAnalysisView.vue` 的局部样式。
- 更新 `frontend-draft-workbench.test.ts`，验证 fixed 定位、双列移动端布局和正文底部留白。
- 不修改 `DraftAssistant.vue`、API 客户端或后端服务。

## 验收标准

- 两个按钮位于同一条固定悬浮栏并左右分布。
- 页面上下滚动时操作栏持续可见。
- 移动端无横向溢出，按钮文字不越界。
- 分析正文末尾不被悬浮栏遮挡。
- 原有事件与禁用状态保持不变。
- 前端回归测试与生产构建通过。
