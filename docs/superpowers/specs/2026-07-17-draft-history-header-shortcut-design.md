# 拟稿助手历史记录快捷入口设计

## 目标

将拟稿助手顶栏左侧的“返回工作台”按钮替换为历史记录快捷入口，让用户在输入、分析、提纲编辑和确认阶段都能随时打开历史编报抽屉。

本次仅调整前端顶栏交互，不修改历史记录接口、草稿数据或后端逻辑。

## 交互

- 顶栏左侧始终显示历史记录图标按钮。
- 点击按钮打开现有 `DraftHistorySidebar`。
- 按钮使用 `查看历史编报` 作为无障碍名称和悬浮提示。
- 任务处于导入阶段时沿用现有禁用规则，避免切换历史记录中断创建流程。
- 删除顶栏右侧原有历史记录按钮，右侧使用等宽占位，保持“拟稿助手”标题居中。
- 不再从拟稿助手顶栏提供“返回工作台”按钮；用户仍可通过顶部模块导航切换功能。

## 组件范围

- 修改 `DraftAssistant.vue` 顶栏模板。
- 删除只为隐藏输入页历史按钮而存在的 `canShowHistory` 计算属性。
- 保留 `historyOpen`、历史抽屉事件和数据加载逻辑不变。

## 验收标准

- 输入页左上角显示历史记录按钮，不显示返回箭头。
- 点击左侧按钮能打开历史编报抽屉。
- 分析、提纲编辑和确认阶段不出现重复历史按钮。
- 顶栏标题在桌面和手机端保持水平居中。
- 现有拟稿助手前端测试与生产构建通过。
