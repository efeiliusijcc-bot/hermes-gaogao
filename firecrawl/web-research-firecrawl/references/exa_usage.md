# Exa API Usage Reference

This document covers the Exa API functions used in the triad intelligence pipeline. The implementation lives in `scripts/exa_client.py`.

---

## Overview

Exa provides semantic search capabilities that complement Tavily's real-time search. While Tavily excels at fresh, news-oriented results, Exa understands meaning and finds semantically relevant content -- research papers, technical analyses, and deep exploratory articles.

The triad pipeline uses Exa alongside Tavily during the search phase to maximize both freshness and depth.

---

## Functions

### exa_search

Semantic search with query autoprompt optimization.

```python
from exa_client import exa_search

result = exa_search(
    query="large language model safety alignment",
    num_results=5,
    use_autoprompt=True,
    include_summary=True,
    category=None,
)
```

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `query` | str | required | Natural language search query |
| `num_results` | int | 5 | Number of results to return |
| `use_autoprompt` | bool | True | Let Exa optimize the query for better semantic matching |
| `include_summary` | bool | True | Include AI-generated summaries for each result |
| `category` | str or None | None | Optional category filter |

**Returns:**

```python
{
    "success": True,
    "mode": "exa_search",
    "input": {"query": "...", "num_results": 5},
    "data": {
        "results": [
            {
                "url": "https://arxiv.org/abs/2024.xxxxx",
                "title": "Paper Title",
                "score": 0.92,
                "summary": "AI-generated summary of the paper..."
            }
        ]
    },
    "error": None
}
```

**When to use:**
- Academic and research queries where semantic relevance matters more than recency.
- Conceptual questions that benefit from understanding meaning, not just keyword matching.
- Finding research papers, technical reports, and in-depth analyses.

### exa_find_similar

Find pages semantically similar to a reference URL.

```python
from exa_client import exa_find_similar

result = exa_find_similar(
    url="https://arxiv.org/abs/2024.xxxxx",
    num_results=5,
    include_summary=True,
)
```

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `url` | str | required | Reference URL to find similar pages for |
| `num_results` | int | 5 | Number of similar results to return |
| `include_summary` | bool | True | Include AI-generated summaries |

**Returns:**

```python
{
    "success": True,
    "mode": "exa_similar",
    "input": {"url": "...", "num_results": 5},
    "data": {
        "results": [
            {
                "url": "https://example.com/related-article",
                "title": "Related Article Title",
                "score": 0.88,
                "summary": "Summary of the related article..."
            }
        ]
    },
    "error": None
}
```

**When to use:**
- Expanding a research direction from a known high-quality source.
- Finding alternative perspectives on the same topic.
- Building a reading list around a key paper or article.

## Category filters

Exa supports category filters to narrow search results by content type.

| Category | Description | Best for |
|----------|-------------|----------|
| `news` | News articles and current events coverage | Recent developments, event coverage |
| `company` | Company pages, about pages, press releases | Corporate information, announcements |
| `research paper` | Academic papers and preprints | Scholarly research, technical depth |
| `linkedin profile` | LinkedIn professional profiles | People search, expertise lookup |
| `github` | GitHub repositories and code | Code, open source projects |
| `tweet` | Twitter/X posts | Social media perspectives |
| `movie` | Movie and entertainment pages | Entertainment content |
| `personal site` | Personal websites and blogs | Individual perspectives |
| `pdf` | PDF documents | Reports, whitepapers, technical docs |

**Usage:**

```python
result = exa_search(
    query="AI regulation policy",
    num_results=5,
    category="research paper",
)
```

**In the triad pipeline:**
- The pipeline does not use category filters by default to maximize coverage.
- When the user specifies a research focus (e.g., "find academic papers on X"), the `category` parameter can be passed through.

## Content options

### Summary

When `include_summary=True` (default), each result includes an AI-generated summary. This summary is:

- Automatically generated by Exa's summarization model.
- Typically 1-3 sentences long.
- Stored in the `summary` field of each result.
- Used by the triad pipeline as the `snippet` field (truncated to 300 chars).

### Text content

For full text extraction, Exa supports a `contents` parameter with text options. The triad pipeline uses summary mode for speed and relies on Firecrawl/Tavily Extract for full content during the scrape phase.

## Integration with triad pipeline

### Search phase

In `intelligence_pipeline.py`, Exa is called alongside Tavily during the search phase of all three modes:

```python
# From intelligence_pipeline.py
e_res = exa_search(query, num_results=max_sources)
e_data = e_res.get("data") if e_res.get("success") else None
```

### Result merging

Exa results are normalized and merged with Tavily results in `_merge_sources()`:

```python
# Exa results are normalized to match the same schema as Tavily
sources.append({
    "url": r.url,
    "title": r.title,
    "snippet": (r.summary or "")[:300],
    "engine": "exa",
    "score": r.score,
})
```

### Fallback behavior

- If Exa fails, the pipeline continues with Tavily results only.
- If Tavily fails, the pipeline continues with Exa results only.
- If both fail, the pipeline returns `success: false`.
- Exa failures are recorded in the `errors` list of the output.

### Credibility scoring

Exa results go through the same `source_ranker.py` credibility scoring as Tavily results. The `engine` field is set to `"exa"` to distinguish the source, but credibility is determined by domain, not by engine.

## Autoprompt

When `use_autoprompt=True` (default), Exa rewrites the user's query into an optimized semantic search query. This is recommended for most use cases because:

- It converts natural language questions into effective search queries.
- It expands abbreviations and clarifies ambiguous terms.
- It improves recall for conceptual queries.

Disable autoprompt (`use_autoprompt=False`) when:

- The query is already a precise, well-formed search string.
- You want exact keyword matching without semantic expansion.
- Testing or debugging query behavior.

When autoprompt is disabled, Exa uses keyword search mode (`type="keyword"`) instead of semantic search (`type="auto"`).

## Error handling

All Exa functions return structured errors in the unified format:

```python
{
    "success": False,
    "mode": "exa_search",
    "input": {"query": "..."},
    "data": None,
    "error": "EXA_API_KEY environment variable is not set."
}
```

Common errors:

| Error | Cause | Fix |
|-------|-------|-----|
| Missing API key | `EXA_API_KEY` env var not set | Set the environment variable |
| Invalid query | Empty or non-string query | Provide a non-empty string |
| Rate limit | API rate limit exceeded | Wait and retry |
| Network error | Connection timeout or DNS failure | Check network connectivity |
